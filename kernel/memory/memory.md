# 内存管理

## 前提

假设已经了解虚拟内存，物理内存， 页等概念。 假如您对下面问题都了解，麻烦指正。

## 问题
+ 应用可以申请比物理内存还大的内存吗（可以的话操作系统会怎么处理）
+ 应用申请一个字节的内存实际上是多少 （申请1M，10M呢）
+ 内存碎片怎么处理 （应用层需要如何配合）
+ 内存映射实现机制
+ 堆空间最大能申请多少
+ 当申请的内存大于当前最大的连续物理内存怎么处理
+ NUMA与UMA对应用的影响
+ 不同内存分配的性能开销

## 内存管理核心目标
+ 易用性（应用开发不需要关心底层内存分配信息）
+ 安全性（防止恶意应用读写其他应用地址）
+ 扩展性（底层架构变化对应用透明，方便各自扩展）
+ 高性能（底层内存分配机制）
+ 反碎片（碎片化会破坏cache，增大管理复杂性）

## 内存管理核心设计
+ 虚拟地址空间管理
+ 虚拟地址与物理地址映射机制
+ 物理内存管理机制
